// HOW IT WORKS
// ============
//
// 1. ON FIRST VISIT:
//    - User sees a cookie consent banner at the bottom of the page
//    - Banner appears after 800ms to avoid being jarring
//    - User can: Accept All, Reject Non-Essential, or Manage Preferences
//    - Banner does NOT appear again until consent changes
//
// 2. CONSENT STORAGE:
//    - User's choice is stored in localStorage (key: "insighted_cookie_consent")
//    - Stored object includes: {necessary: true, analytics: false, functional: false, marketing: false}
//    - No expiration - persists across browser sessions
//    - User can change preferences anytime via footer "Cookie Preferences" link
//
// 3. SCRIPT LOADING:
//    - Google Analytics: Loads ONLY if analytics consent = true
//    - Microsoft Clarity: Loads ONLY if analytics or functional consent = true
//    - Marketing pixels: Loads ONLY if marketing consent = true
//    - Necessary cookies: Always active (no consent needed)
//
// 4. TRANSPARENCY:
//    - Essential/Necessary cookies: Always ON, cannot be disabled
//    - Analytics: Can be enabled/disabled
//    - Functional: Can be enabled/disabled  
//    - Marketing: Can be enabled/disabled
//    - User sees clear descriptions of each category
//
//
// COMPLIANCE FEATURES
// ====================
//
// ✓ Explicit Opt-in: Non-essential cookies OFF by default
// ✓ No Dark Patterns: Reject button is as prominent as Accept
// ✓ Granular Control: Users can customize each category
// ✓ Withdraw Consent: Footer link allows users to change settings anytime
// ✓ Transparency: Clear descriptions of cookie purposes
// ✓ GDPR Ready: Respects strict consent requirements
// ✓ CCPA Ready: Provides opt-out mechanism
// ✓ Accessibility: Keyboard navigation, focus management, ARIA labels
// ✓ Mobile Responsive: Works on all device sizes
// ✓ Performance: No external dependencies, vanilla JS only
//

// TROUBLESHOOTING
// ===============
//
// Q: Banner not appearing on first visit?
// A: Check browser console for errors. Ensure scripts are loaded in order:
//    1. script src="script.js" (existing page script)
//    2. script src="cookie-consent.js" (cookie consent)
//    Then the banner HTML markup
//
// Q: Console errors about "getElementById"?
// A: Make sure the banner and modal HTML markup is in your <body>
//    Check that IDs match: id="cookie-consent-banner", id="cookie-preferences-modal", etc.
//
// Q: Google Analytics not tracking after accepting consent?
// A: Check DevTools > Network tab for googletagmanager.com requests
//    If missing: Clear localStorage, decline consent, reload
//    Then accept again and watch for the request
//
// Q: Modal not opening when clicking "Cookie Preferences"?
// A: Check that the footer link has id="cookie-preferences-link"
//    Ensure cookie-consent.js is loaded AFTER the markup
//
// Q: Styling looks wrong?
// A: Verify cookie-consent.css is linked in <head>:
//    <link rel="stylesheet" href="cookie-consent.css">
//    Check for CSS conflicts with your main styles.css
//
// Q: Unnecessary delay when loading pages?
// A: The banner appears after 800ms to avoid jarring. Can be adjusted in:
//    cookie-consent.js, setTimeout(() => { showBanner(); }, 800);
//    Change 800 to smaller/larger number in milliseconds
